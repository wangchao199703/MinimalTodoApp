## v1.1.4 —— 2026-06-04

界面一致性 / 可读性修复 + 新增 12 套配色主题（对应需求 1–13）。

- 体积（需求1）：WPF 自包含单文件无法安全裁剪，按"无依赖"约束走极保守优化，体积约 ~60MB。
- 子待办计数即时刷新；"所有待办"显示已完成子待办，与普通分组一致（需求2）。
- 右键菜单去除左侧白色矩形（自定义无 gutter 模板）（需求3）。
- 左侧分组层级缩进：普通分组缩进为"所有待办"子级（需求4）。
- 精简任务右键菜单：移除 清除截止日期 / 折叠展开 / 切换周期提醒 / 标记完成（需求5）。
- 支持拖动任务到左侧分组移动归属（需求6）。
- 编辑任务弹窗可拖动（需求7）。
- 所有窗体/弹层圆角统一（补充圆角 ToolTip / ComboBox）（需求8）。
- 1080p 清晰度：Per-Monitor V2 DPI 清单 + 像素对齐文本渲染（需求9）。
- 缩小添加按钮（需求10）。
- 设置新增"关于"分组：版本号 + 检查更新 + GitHub 链接（需求11）。
- 新增 12 套主题：莫兰迪×3 / 马卡龙×3 / 敦煌×3 / 蒙德里安×3（需求12）。
- 各主题弹窗/预览文字对比度校验，深色/纯黑下可读（需求13）。

版本号同步：`<Version>1.1.4` / `<FileVersion>1.1.4.0` / `<AssemblyVersion>1.1.4.0`。

### 主题独立窗口 + 分组扩充 + 圆角修复（v1.1.4，2026-06-07，本地构建·待发布）

> 版本号保持 **1.1.4**（`<Version>` / `<FileVersion>` / `<AssemblyVersion>` 不变）；本次仅推送代码与本地无依赖 exe，暂不发 Release。详细实现见 `优化记录.md` 第三十二轮。

- **修改主题改为独立窗口**：新增 `ThemePickerWindow`，可拖动、定位主程序侧边不遮挡、单击主题即实时预览并持久化；「新建自定义主题」改为非模态。
- **主题分组 + 每组补满 12 套**：常用（当前 + 最近使用 9 个，按上次使用排序）、经典、莫兰迪、马卡龙、敦煌、蒙德里安、孟菲斯、洛可可、马蒂斯（命名组各 12 套）、透明（透明/毛玻璃 + 磨砂×2 + 薄雾）。配色按对比度锚定派生，深色主题下右键菜单与日历均可读。
- **圆角修复**：关闭 `ContextMenu`/`ToolTip` 的系统方角投影（右键菜单不再露直角）；主题化并圆角化系统 `DatePicker`/`Calendar`。
- **修坑**：`ThemeManager.BuildFromColors` 补齐 `PopupBg`，修复调色板/自定义主题弹窗背景串色。
- 数据兼容：仅新增 `AppData.ThemeUsageOrder`（默认空），向后兼容。

### 补丁（同 1.1.4 / 1.1.3 资产，2026-06-04）

- 修复"检查更新"在 GitHub 匿名接口限流(403)等失败时误报"已是最新"：`CheckAsync` 失败改抛异常，手动检查显示"检查失败，请稍后重试"。
- 检查频率由每 1 小时放宽到**启动后一次 + 每 12 小时一次**，降低撞限流概率。
- 以修复后的构建替换 **v1.1.4** 与 **v1.1.3** 两个 Release 的 exe 资产（版本号分别保持 1.1.4 / 1.1.3）。

### 补丁二（同 1.1.4 / 1.1.3 资产，2026-06-04）

- 更新对话框的发布说明**跟随用户界面语言**只显示对应语种（中/英），不再中英混排。
- 发布说明把原始 Markdown 渲染为**可读文本**（去掉 `#`、`**`、列表记号等）。
- 修复**点更新后旧版退出、新版未启动**：更新重启(带 `--updated-from`)时单实例静默接管不弹框、接管超时也继续运行；启动新版改为带重试 + 指定工作目录，规避新 exe 刚落地被占用导致拉起失败。

### 补丁三（同 1.1.4 / 1.1.3 资产，2026-06-04）

- **彻底修复「点更新后新版在部分电脑上仍不启动」**：根因是上一版的重启依赖隐藏 PowerShell 临时脚本，在受控/企业机器（组策略把 ExecutionPolicy 设在 MachinePolicy 作用域使 `-Bypass` 失效、AppLocker/杀软拦截 `%TEMP%` 脚本、Constrained Language Mode）会**静默失败**，而旧版此时已退出，用户毫无察觉。
  - 改为**直接 `Process.Start`（`UseShellExecute=true`）拉起新版**，不再依赖 PowerShell/cmd 脚本（`UpdateService.TryStartNewVersion`）。
  - 新增**「新版已启动」命名事件握手**：旧版拉起新版后等待确认，新版启动即 Set（`SignalUpdatedStarted`）。
  - **兜底提醒**：若多次尝试仍未确认新版启动，旧版**保持存活**并弹出对话框，显示新版完整路径与「打开所在文件夹 / 立即运行 / 关闭」，引导用户手动运行。
- 以修复后的构建替换 **v1.1.4** 与 **v1.1.3** 两个 Release 的 exe 资产（版本号分别保持 1.1.4 / 1.1.3）。
