# MinimalTodoApp 版本发布记录

> 每次发布在此登记版本号与变更摘要（最新在上）。详细实现见同目录 `优化记录.md`，对外发布说明见仓库根 `release-notes.md`。

## v1.1.7（第三十六轮）—— 2026-06-10

版本号升至 **1.1.7**（`<Version>1.1.7` / `<FileVersion>1.1.7.0` / `<AssemblyVersion>1.1.7.0`）。界面现代化 + iOS/macOS 风格灵动动画（**本轮仅构建无依赖 exe，暂不发布 Release 包**）：

1. **统一动画规范**：新增 `Anim` 动画工具（Fast 120ms / Base 200ms / Emphasis 280ms 弹簧过冲 / Bounce 弹跳），全部动画跟随 Windows 系统"显示动画"无障碍开关。
2. **按钮按压回弹**：主按钮/幽灵按钮/图标按钮按下轻缩、松开弹簧回弹；hover 由瞬时变色改为平滑渐变。
3. **勾选对勾弹跳**：完成勾选时对勾 0→1.18→1 过冲回弹 + 淡入，与完成滑出动画衔接（先弹勾再飞走）。
4. **任务卡片浮起**：hover 平滑渐变 + 轻阴影"浮起"；卡片/输入框内边距微调更透气（12,8→14,9 / 10,7→11,8）。
5. **弹窗进场动画**：设置/编辑任务/图标选择/帮助/更新/主题选择/主题编辑 7 个弹窗打开时缩放 0.96→1（弹簧）+ 淡入。
6. **新任务出现动画**：添加待办后新卡片淡入 + 弹性上移。
7. **托盘恢复淡入 / 日程面板弹性滑入 / Toast 弹簧上滑**。
8. **行距 bug 修复**：设置滑块拖到 0.4 不再被 FontManager 夹回 0.8；非法值兜底对齐产品默认 13/1.0。
9. **全主题适配**：所有外观/动画改动仅使用既有 18 个主题颜色键，33+ 套内置主题与自定义主题自动适配；`AppData` 无字段变更、零新增文案（中英文资源无改动）。

> 详细实现见 `优化记录.md` 第三十六轮。

## v1.1.6（第三十五轮）—— 2026-06-08

版本号升至 **1.1.6**（`<Version>1.1.6` / `<FileVersion>1.1.6.0` / `<AssemblyVersion>1.1.6.0`）。8 项体验反馈修复（**本轮仅构建无依赖 exe，暂不发布 Release 包**）：

1. **日历多任务并排**：天/周视图同时段任务按碰撞分簇平分列宽并排、不再重叠；月视图改为横向并排紧凑色块（自动换行 + `+N`）。
2. **日期选择器全皮肤适配**：新增 `CalendarDayButton`/`CalendarButton` 主题化模板，所有深色/彩色皮肤下日号、今天、选中、非本月均清晰可辨。
3. **拖任务到日历设截止时间**：从待办列表拖任务到日历即设截止时间——天/周到落点对应整点、月到当天 18:00；首次打开日历弹一次功能提示（仅一次）。
4. **背景调柔**：马卡龙（薄荷/樱花/柠檬）与敦煌（石青/土红）等鲜艳浅色主题整体降饱和、更耐久看。
5. **行距下限放宽**：设置里行距最小值由 0.8 降到 0.4，可更紧凑。
6. **默认排版优化**：默认字号 13、行距 1.0、勾选框 18（更舒适，仅影响新安装/恢复默认）。
7. **国内节假日**：日历联网显示国内法定节假日（标注节日名 + 放假日高亮），设置中可开关、默认开启。
8. **关于页改为「支持开发者」**：邀请到 GitHub 给项目点 Star，点击打开仓库。

> 详细实现见 `优化记录.md` 第三十五轮。

## v1.1.5（重新发布·第三十四轮）—— 2026-06-07

版本号**保持 1.1.5**（`<Version>1.1.5` / `<FileVersion>1.1.5.0` / `<AssemblyVersion>1.1.5.0` 未变），删除原 `v1.1.5` tag 与 Release 后用本轮构建覆盖重发资产。6 项体验反馈修复：

1. **托盘显示主界面取消贴边并居中**：贴边隐藏时点托盘「显示」先解除贴边，再居中到光标所在屏。
2. **日历缩放只缩日历**：开日历后拖窗口外边缘只缩放日历、待办宽度不变；待办宽度仅由中间分隔条调整。
3. **主题收藏分组**：任意主题右键可加入/取消收藏，「收藏」分组**置于最前**（在「常用」之上），可拖动排序（持久化 `AppData.FavoriteThemeKeys`）；分组始终显示，为空时展示「右键收藏」提示文字。
7. **使用说明刷新**：应用内「使用说明」补全历史+本轮全部功能（独立主题窗口/收藏/实时预览/日历缩放/托盘居中/自动更新）。
4. **自定义主题日历兼容**：自定义主题的日历相关辅助色由已编辑色派生，今日高亮/悬停/次要文字随主题协调。
5. **自定义主题实时预览**：编辑改色即时换肤（含日历），未保存关闭还原旧主题，保存才持久化。
6. **圆角排查**：审计全部弹窗均已透明圆角；顶层右键菜单 Border 增加阴影留白边距。

> 注意：版本号未变，已在 1.1.5 的用户不会被自动更新提示到本轮修复。详细实现见 `优化记录.md` 第三十四轮。

### 主题窗口反馈修复（v1.1.4，2026-06-07，本地构建·待发布）

> 版本号保持 **1.1.4** 不变。基于上一轮实测反馈的 6 项修复；详见 `优化记录.md` 第三十三轮。

- **右键菜单直角**：撤销上一轮 `HasDropShadow=False`（它会关掉弹窗透明导致四角被填成方角）。
- **深色主题日历可读性**：根因是日历用代码画刷快照渲染、切主题不重绘；新增 `ThemeManager.ThemeChanged` 事件让日历随主题实时重渲染，并提升星期表头对比度。
- **自定义主题可编辑/删除**：色板右键菜单「编辑/删除」；编辑器新增编辑模式（保留 Key 与未编辑颜色）。
- **主题窗口可调整大小**：`WindowChrome` 可 resize + 圆角裁剪随尺寸更新；色板随窗口拉宽显示更多。
- **分组快速选择栏**：顶部分组 chips，点击直接滚动到对应分组（非下拉框）。
- **自定义主题改“选颜色”**：新增纯 WPF 的 HSV 取色器（色盘 + 色相滑条 + 十六进制），替换手输颜色。

### 补丁（同 1.1.4 / 1.1.3 资产，2026-06-04）

- 修复"检查更新"在 GitHub 匿名接口限流(403)等失败时误报"已是最新"：`CheckAsync` 失败改抛异常，手动检查显示"检查失败，请稍后重试"。
- 检查频率由每 1 小时放宽到**启动后一次 + 每 12 小时一次**，降低撞限流概率。
- 以修复后的构建替换 **v1.1.4** 与 **v1.1.3** 两个 Release 的 exe 资产（版本号分别保持 1.1.4 / 1.1.3）。

### 补丁二（同 1.1.4 / 1.1.3 资产，2026-06-04）

- 更新对话框的发布说明**跟随用户界面语言**只显示对应语种（中/英），不再中英混排。
- 发布说明把原始 Markdown 渲染为**可读文本**（去掉 `#`、`**`、列表记号等）。
- 修复**点更新后旧版退出、新版未启动**：更新重启(带 `--updated-from`)时单实例静默接管不弹框、接管超时也继续运行；启动新版改为带重试 + 指定工作目录，规避新 exe 刚落地被占用导致拉起失败。

### 补丁三（同 1.1.4 / 1.1.3 资产，2026-06-04）

- **彻底修复「点更新后新版在部分电脑上仍不启动」**：根因是上一版的重启依赖隐藏 PowerShell 临时脚本，在受控/企业机器（组策略把 ExecutionPolicy 设在 MachinePolicy 作用域使 `-Bypass` 失效、AppLocker/杀软拦截 `%TEMP%` 脚本、Constrained Language Mode）会**静默失败**，而旧版此时已退出，用户毫无察觉。
  - 改为**直接 `Process.Start`（`UseShellExecute=true`）拉起新版**，不再依赖 PowerShell/cmd 脚本（`UpdateService.TryStartNewVersion`）。
  - 新增**「新版已启动」命名事件握手**：旧版拉起新版后等待确认，新版启动即 Set（`SignalUpdatedStarted`）。
  - **兜底提醒**：若多次尝试仍未确认新版启动，旧版**保持存活**并弹出对话框，显示新版完整路径与「打开所在文件夹 / 立即运行 / 关闭」，引导用户手动运行。
- 以修复后的构建替换 **v1.1.4** 与 **v1.1.3** 两个 Release 的 exe 资产（版本号分别保持 1.1.4 / 1.1.3）。
