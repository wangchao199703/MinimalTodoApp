## 各项详细说明

### 1. 常驻桌面 + 托盘
- 标题栏右侧只有「最小化」和「关闭」两个键，**关闭键不退出程序**，而是 `Hide()` 到系统托盘。
- 程序真正退出的唯一入口：**托盘图标右键 → 退出**（`ExitMenuItem_Click` 中置 `_allowClose=true` 后 `Shutdown`）。
- 托盘双击 / 右键「显示主界面」可恢复窗口。
- 相关文件：`Views/MainWindow.xaml.cs`（`OnClosing`、`HideToTray_Click`、`ExitMenuItem_Click`）。

### 2. 分组栏宽度调整 / 隐藏 / 详情
- **调宽**：主体 Grid 左列 + `GridSplitter`，拖动改变 `SidebarColumn.Width`，关窗时写回 `AppData.SidebarWidth` 持久化。
- **隐藏/展开**：标题栏「☰」按钮触发 `ToggleSidebarCommand`，代码联动把左列宽度设为 0 或恢复持久化宽度；`SidebarCollapsed` 也持久化。
- **分组详情**：每个分组项右侧「ⓘ」`ToggleButton` 控制一个 `Popup`，显示分组名、颜色、未完成数量，并提供「删除该分组」。再次点击或点击别处即收起。

### 3. 新任务日期按需弹出
- 底部添加栏左侧「📅」按钮 → `ToggleNewTaskDateCommand` 切换 `IsNewTaskDatePickerOpen`。
- 仅当展开时 `DatePicker` 才可见；收起时自动清空 `NewTaskDueDate`，保证“平时无日期框”。

### 4. 新建分组按需弹出
- 默认显示「＋ 新建分组」按钮（`BeginAddGroupCommand` → `IsAddingGroup=true`）。
- 展开后显示输入框 + 「✓ 确定」「✕ 取消」；回车确定、Esc 取消。
- 命令：`ConfirmAddGroupCommand` / `CancelAddGroupCommand`。

### 5. 添加任务移到底部
- 右侧区域改为三行：标题/排序（顶）、任务列表（中，占满）、添加栏（底）。

### 6. 倒计时 + 右键操作
- `TodoItem` 新增计算属性：`DueState`（枚举）、`DueCountdownText`（“今天到期 / 明天到期 / 剩 N 天 / 已逾期 N 天 / 已完成”）、`HasDueDate`。
- 倒计时颜色由 `DueStateToBrushConverter` 按状态映射主题色（逾期红、临近橙、正常次要色）。
- `DispatcherTimer` 每分钟刷新一次，跨零点也能自动更新文案。
- 任务行**不再有日期选择框**；右键菜单提供：设置/修改截止日期…（弹 `DueDateDialog`）、清除截止日期、标记完成、删除任务。
- ContextMenu 因不在可视树，命令通过 `PlacementTarget.Tag`（Tag 绑定到窗口 DataContext）访问 ViewModel。

### 7. 自定义窗口样式
- `WindowStyle="None"` + `System.Windows.Shell.WindowChrome`：去掉系统标题栏，保留拖动、缩放、贴边、任务栏行为。
- 自绘 40px 标题栏（`TitleBarBg` 主题色），与窗体一致；标题栏按钮用 `Segoe MDL2 Assets` 图标字体。
- 按钮加 `WindowChrome.IsHitTestVisibleInChrome=True` 以便在标题区可点击。

### 8. 多套主题
- `ThemeManager` 泛化为支持任意多套主题（`ThemeManager.All`）。
- 主题文件均在 `Themes/`，键名完全一致，新增统一键 `TitleBarBg`、`WarningText`、`SuccessText`。
- 6 套：`Light` 明亮、`Dark` 暗黑、`Nord` 极地、`Ocean` 海洋、`Forest` 森林、`Rose` 玫瑰。
- 侧边栏底部「🎨」下拉切换，选择即时生效并持久化（`AppData.Theme`）。

### 9. 打包
- 命令（框架依赖，单文件，体积最小，目标机需 .NET 8 桌面运行时）：
  ```powershell
  dotnet publish MinimalTodoApp\MinimalTodoApp.csproj -c Release -r win-x64 `
    --self-contained false -p:PublishSingleFile=true -p:DebugType=none
  ```
- 产物：`bin\Release\net8.0-windows\win-x64\publish\MinimalTodoApp.exe`，**约 1.68 MB**，单文件可直接拷贝运行。
- 若目标机没有 .NET 8 运行时，改用自包含（体积约 70–90MB）：
  ```powershell
  dotnet publish MinimalTodoApp\MinimalTodoApp.csproj -c Release -r win-x64 `
    --self-contained true -p:PublishSingleFile=true -p:EnableCompressionInSingleFile=true
  ```

- 模型新增 `Priority`（None/Low/Medium/High）。

- 倒计时定时器从每分钟改为**每 30 秒**刷新。

### 6. 右键编辑（修复 + 增强）
- **修复**：旧版右键「设置截止日期」依赖的菜单项解析在 ContextMenu 场景下不稳定；改为优先用 `MenuItem.DataContext`（ContextMenu 继承 PlacementTarget 的 DataContext）直接拿到任务，并保留 PlacementTarget 兜底。

- 右键菜单：编辑(优先级/截止时间)… / 清除截止日期 / 标记完成 / 删除任务。
- ViewModel 新增 `ApplyTaskEdits(item, due, priority)` 统一回填。

## 总览
| # | 需求 | 状态 | 关键实现 |
|---|------|------|----------|
| 1 | 侧边栏展开时，图标与图标对齐、所有文字左边也对齐 | ✅ | 每个侧栏项的图标统一放进 `Width=20 + TextAlignment=Center` 的固定列，文字统一 `Margin=8,0,0,0`；分组色点也用同宽列 |
| 2 | 「新建分组」图标+文字太宽，与下方「使用说明」保持一致，并置于其上 | ✅ | 「新建分组」改用与「使用说明」相同的图标列模板，顶部间距 `8→2`；本就位于「使用说明」上方 |
| 3 | 标题栏右键「置于顶层」无效 | ✅ | 改由代码托管 `Topmost`，修复 `ShowMainWindow` 置顶技巧破坏绑定的问题 |
| 4 | 添加任务时光标与输入文本未靠左 | ✅ | `FlatTextBox` 显式 `HorizontalContentAlignment/TextAlignment=Left`，占位文字 `Margin` 对齐到 10 |

## 详细说明

### 1. 侧边栏图标 / 文字对齐
- 问题：各侧栏项图标字形宽度不一（emoji 📋📖🎨 偏宽、符号 ＋ ☰ 偏窄），且「全部任务」图标未设 `FontSize`，导致文字起始位置参差不齐。
- 方案：每个图标 `TextBlock` 统一 `FontSize=15 Width=20 TextAlignment=Center`，构成固定宽度的「图标列」；文字统一 `Margin=8,0,0,0`。
  这样无论字形宽窄，文字都从同一 x 起始（左内边距 8 + 图标列 20 + 间距 8）。
- 分组列表项：色点 `Ellipse` 列宽由 `Auto` 改为 `20` 并 `HorizontalAlignment=Center`，分组名加 `Margin=8,0,0,0`，与上方导航项文字对齐。
- 涉及：`Views/MainWindow.xaml`（全部任务 / 分组项 / 新建分组 / 使用说明 / 修改主题 / 收起侧边栏）。

### 2.「新建分组」与「使用说明」一致

### 1. 自动贴边功能修复

#### 问题分析
- **左右贴边失效**：原代码已有左右贴边检测逻辑，但触发区域判断有问题
- **上边触发区域过窄**：隐藏态下仅在窗口横向范围内才触发，应该整个屏幕上边缘都能触发
- **误隐藏问题**：显示态下缓冲区仅 6px，鼠标稍微移动就触发隐藏
- **Windows手势冲突**：拖到边缘触发系统的 Aero Snap 自动最大化/分屏

#### 修复方案
1. **触发区域优化**（`MainWindow.xaml.cs#EdgeProbeTick`）：
   - 上边：去掉横向范围限制，鼠标到达屏幕上边缘任意位置即触发
   - 左边：去掉纵向范围限制，鼠标到达屏幕左边缘任意位置即触发
   - 右边：去掉纵向范围限制，鼠标到达屏幕右边缘任意位置即触发
   - 触发条件从 `pos.Y >= wa.Top - 1 && pos.Y <= wa.Top + 2 && pos.X >= Left && pos.X <= Left + Width` 改为 `pos.Y >= wa.Top - 1 && pos.Y <= wa.Top + VisibleStripPx + 2`

2. **缓冲区增大**：显示态下的缓冲区从 6px 增大到 20px，避免鼠标在窗口内轻微移动就触发隐藏

3. **禁用 Aero Snap**（`NativeMethods.cs` + `MainWindow.xaml.cs#OnLoaded`）：
   - 新增 `DisableAeroSnap` / `EnableAeroSnap` 方法，通过移除 `WS_MAXIMIZEBOX` 窗口样式禁用系统贴边手势

### 3. 父子待办逻辑优化

#### 数据模型增强
- `TodoItem` 新增 `ParentId` 字段（`Guid?`），建立明确的父子关系
- 原有的 `IndentLevel` 仅用于视觉缩进，`ParentId` 用于逻辑关联

#### 父完成 → 子自动完成
- `MainViewModel#OnItemPropertyChanged`：父待办完成时调用 `CompleteChildren` 递归完成所有子待办
- `CompleteChildren` 方法：查找所有 `ParentId == parent.Id` 的子待办，设置 `IsCompleted = true`，并递归处理子待办的子待办

#### 子完成 → 只放烟花
- `MainWindow#OnTaskCompleting`：判断 `item.ParentId.HasValue`，如果是子待办则：
  - 播放烟花和音效（如果设置开启）
  - **不播放**滑出消失动画
  - 直接调用 `FinishCompletion` 移入"已完成"分组
  - 调用 `CheckParentCompletion` 检查父待办是否应自动完成

#### 全部子完成 → 父自动完成
- `MainViewModel#CheckParentCompletion`：
  - 查找父待办的所有子待办（`ParentId == parent.Id`）
  - 如果所有子待办都已完成（`siblings.All(s => s.IsCompleted)`），自动完成父待办
  - 递归检查父待办的父待办，支持多层嵌套

#### 动画适配
- 父待办完成：播放完整的"滑出 + 淡出 + 行收起"动画 + 烟花特效
- 子待办完成：仅播放烟花特效，不播放消失动画（视觉上保持在列表中，等父待办完成时一起消失）

### 实现要点

#### 1. 移除语音输入

- Win+H 在不同 Windows 设置 / 窗口焦点状态下行为不稳定，按 `SendInput` 模拟仍存在”按下没反应”的场景，故按题目要求直接移除该功能：

#### 2. 父子关系联动 & 整族消失动画

- **父子关系自动推断**：以 `IndentLevel` 与同分组内的 `OrderIndex` 顺序为唯一来源，每次”任何会变动层级或顺序”的操作后立即调用 `MainViewModel.RecomputeParents()`：
  - `ChangeIndent`（Tab / Shift+Tab / 右键缩进）
  - `AddTask`（新建任务）
  - `DropTask`（拖拽排序）
  - `ImportMarkdown`（导入）
  - 启动时一次性重算，修正旧数据。
  - 算法：分组内按 `OrderIndex` 顺序遍历，维护一个”层级栈”，遇到一项时弹出 ≥ 自己层级的项，栈顶即为父；`IndentLevel=0` 时 `ParentId=null`。
- **完成行为**（`MainViewModel.OnItemPropertyChanged`）：
  - 子待办勾选：若其父存在且未完成，则**只打钩、不移动到”已完成”分组**，并调用 `CheckParentCompletion` 检查父待办；
  - 父或独立待办勾选：调用 `MarkDescendantsCompletedSilently` 静默把所有后代设为完成（`_completingFamily` 标志抑制重入），随后抛出 `TaskCompleting` 事件给视图层；
  - 当父待办的所有直接子待办均已完成 → 自动把父待办勾上（递归触发上面的”父分支”，从而触发整族消失动画）。
- **整族消失动画**（`MainWindow.xaml.cs#OnTaskCompleting`）：
  - 收集 `Vm.CollectFamily(parent)`（父 + 所有递归子）；
  - 对每个仍在当前视图中的容器同时播放原有 1.1s 的”向左滑出 + 淡出 + 行高收起”动画，并叠加烟花特效；
  - 全部动画结束后回调 `Vm.FinishFamilyCompletion(parent)`，把整族一次性搬到”已完成”分组并刷新列表。
- 关键文件：`Models/TodoItem.cs`（`ParentId / IsCollapsed / HasChildren`）、`ViewModels/MainViewModel.cs`（`RecomputeParents / MarkDescendantsCompletedSilently / CollectFamily / FinishFamilyCompletion`）、`Views/MainWindow.xaml.cs#OnTaskCompleting`。

### 实现要点

#### 1. 右侧贴边隐藏（DPI 修复）

- 原因诊断：贴边判定 `Left + Width >= wa.Right - SnapThresholdPx` 中，`Left/Width` 是 WPF DIP，而 `wa.Right` 来自 `MONITORINFO.rcWork` 物理像素。100% DPI 两者相等，>100% 时 `wa.Right` 远大于 `Left + Width`，条件永远不成立，因此“右侧贴边”从未生效；上 / 左只比 `wa.Top/Left=0`，恰好两套坐标系都是 0，所以一直正常。
- 修复（`Infrastructure/NativeMethods.cs`）：
  - 新增 `ToDip(Rect/Point, Visual)`，利用 `PresentationSource.CompositionTarget.TransformFromDevice` 把物理像素转换为 DIP；
  - 新增 `GetCursorScreenWorkAreaDip(Visual)` / `GetCursorPointDip(Visual)`，与 WPF Left/Top 同坐标系。
- `Views/MainWindow.xaml.cs` 中 `RestoreDockOnLoad`、`TryDockAfterDrag`、`DockTo`、`EdgeProbeTick` 全部改用 DIP 版本，三条边的判定 / 探针都在同一坐标系下比较。

#### 2. 侧边栏图标 / 文字尺寸统一

- 现在统一为：
  - 按钮 Height=28，Padding `6,0`；
  - 所有图标 `FontSize=13`、`Width=18`、水平居中；
  - 所有文字 `FontSize=12`；
  - 分组圆点 12×12 → 10×10；
  - 折叠态 `CollapsedRailWidth` 52 → 39。
- 标题栏按钮也按比例收紧：`CaptionButton` 44×32 → 36×26、`CaptionToggleButton` 40×32 → 32×26。
