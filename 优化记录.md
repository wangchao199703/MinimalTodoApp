## 第 1 次优化（2026-05-30）

### 原始提示词

> 我之前让你写了个待办项目，帮我完成以下需求，然后记录到优化记录.md（记录下原来的提示词），并且输出无依赖的 exe
>
> 1. 支持拖动隐藏，像 qq 一样，拖到到屏幕上面，会播放一个向上的动画，自动隐藏，鼠标移动到屏幕上面，会播放一个动画，app 会自己下来；屏幕左边和右边同理，拖动过去自动隐藏
> 2. 支持待办创建子待办，通过缩进区分不同层级的待办
> 3. 待办支持语音输入，调用 Windows 系统的语音输入接口就行，在输入框右边放个麦克风图标，点击输入即可
> 4. 支持导出 md 功能，在做下角的主题功能上面放一个按钮导入导出，支持导出 md，导入 md（导出图标+文字：导入导出，和之前一样的长度宽度，对齐）
> 5. 支持周期提醒功能，在创建待办的时候新增周期提醒选项，勾选后会周期提醒，默认会有声音和左下角的通知提醒，可以在设置里面调整周期提醒是声音是否开启

### 实现要点

#### 1. QQ 式贴边自动隐藏

- 拖动窗口时若标题栏到达屏幕上/左/右边的 14px 范围内，自动贴边并播放 220ms 滑出动画，仅在边缘保留 4px 触发条，下次启动持久化。
- 隐藏后由 90ms 探针 (`DispatcherTimer`) 轮询光标位置：进入触发条范围立即播放滑入动画弹出；离开窗口区域 6px 缓冲外又自动滑回。
- 多屏场景下记录贴附时的工作区 (`_dockedWa`)，避免光标移到其它屏幕造成误判。
- 再次按住标题栏拖动即可解除贴边。
- 关键文件：`Views/MainWindow.xaml.cs`（`DockTo / HideToEdge / ShowFromEdge / EdgeProbeTick`），`Infrastructure/NativeMethods.cs`（`GetCursorPos / MonitorFromPoint`）。

#### 2. 子待办（缩进表示层级）

- `TodoItem` 新增 `IndentLevel`，最多 6 级。
- 任务模板通过 `IndentToMarginConverter`（每级 22px）转为左外边距，无需嵌套集合即得到层级视觉。
- 操作方式：选中任务后按 `Tab` 增加缩进、`Shift+Tab` 减少缩进；右键菜单亦提供同样命令；导入导出 Markdown 会自动按缩进保留层级。
- 关键文件：`Models/TodoItem.cs`、`Converters/Converters.cs`、`Views/MainWindow.xaml`（任务模板）、`Views/MainWindow.xaml.cs#TaskList_PreviewKeyDown`。

#### 3. 语音输入（Windows 系统语音听写）

- 新任务输入框右侧新增🎤按钮，点击后焦点切到文本框并通过 P/Invoke 模拟 `Win+H`，调起 Windows 11/10 内置“语音输入”，识别结果直接进入文本框。
- 全程无第三方语音 SDK，符合“无依赖 exe”目标。
- 关键文件：`Infrastructure/NativeMethods.cs#InvokeWindowsSpeechInput`、`Views/MainWindow.xaml.cs#VoiceInput_Click`。

#### 4. 导入 / 导出 Markdown

- 侧边栏在“修改主题”上方新增📤“导入导出” `ToggleButton`，与“使用说明”等高（34px）、同字号、同对齐方式（展开左对齐、折叠居中）；点击弹出气泡菜单：「导出为 Markdown」「从 Markdown 导入」。
- 导出格式：`## 分组名` + `- [ ] / - [x] 标题` + 可选「（截止 …）」尾注；子待办用每级 2 个空格的缩进；与主流 Markdown 编辑器互通。
- 导入：识别二级及以上标题作为分组名、解析复选框状态与缩进，自动归类到对应分组（不存在时新建）。
- 关键文件：`Infrastructure/MarkdownService.cs`、`ViewModels/MainViewModel.cs#BuildMarkdown/ImportMarkdown`、`Views/MainWindow.xaml.cs#ExportMarkdown_Click/ImportMarkdown_Click`。

#### 5. 周期提醒

- 新任务输入区新增「周期提醒」勾选 + 间隔下拉（5/10/15/30/60/120 分钟，默认 30）。
- `TodoItem` 新增 `ReminderEnabled / ReminderIntervalMinutes / LastRemindedAt`；`MainViewModel` 的 30s 倒计时定时器顺带触发 `CheckReminder`，达到间隔即抛出 `ReminderTriggered` 事件。
- 视图层订阅事件：弹出左下角 `ToastWindow`（淡入 + 上滑 220ms + 5s 自动淡出，可堆叠）、按设置播放轻提示音（内存合成的 1175Hz 马林巴音色，无外部音频文件）。
- 设置对话框新增「周期提醒声音」开关，默认开启；关闭后仅显示左下角通知，不发声。
- 任务列表条目末尾展示🔔小图标提示当前任务已开启周期提醒；右键菜单可随时切换。
- 关键文件：`Models/TodoItem.cs`、`Models/AppData.cs`（`ReminderSoundEnabled / DockEdge`）、`ViewModels/MainViewModel.cs#CheckReminder`、`Views/ToastWindow.xaml(.cs)`、`Infrastructure/ReminderSound.cs`、`Views/SettingsDialog.xaml(.cs)`。

### 兼容性 & 体积

- 仍使用现有的 `gong-wpf-dragdrop` + `H.NotifyIcon.Wpf` + `CommunityToolkit.Mvvm` 三个包，未新增依赖；语音 / 屏幕信息全部使用纯 P/Invoke 实现，故未引入 `UseWindowsForms`。
- 单文件 self-contained 发布产物 ~63 MB（启用单文件压缩 + 仅英文卫星资源 + 去调试符号），可在干净的 Windows 上免装 .NET 8 运行时直接运行。
- 构建命令：`build.bat` 或 `build.ps1`（参数见脚本）。

### 输出

- 单文件 exe 路径：`MinimalTodoApp/bin/Release/net8.0-windows/win-x64/publish/MinimalTodoApp.exe`

## 第 2 次优化（2026-05-30）

### 原始提示词

> 我之前让你写了个待办项目，帮我完成以下需求，然后记录到优化记录.md（记录下原来的提示词），并且输出无依赖的 exe
>
> 1. 语音输入功能还是无效，修复一下，如果修复不了，移除语音输入功能
> 2. 右键修改优先级，未更新父子关系。新增和修改的关系都要更新父子关系（父有两个子待办，完成一个子待办，不消失，只是打钩完成，两个子完成，父待办也自动完成，消失所有父子待办，放消失动画和烟花）
> 3. 父待办支持折叠，折叠后隐藏子待办
> 4. 贴边隐藏功能，鼠标在 app 上时，一直闪屏，要求的效果是：鼠标在 app 上移动，app 不隐藏，可以修改；鼠标移开 app 时，自动回到原来隐藏的地方
> 5. 优先级默认为中，取消无优先级选项

### 实现要点

#### 1. 移除语音输入

- Win+H 在不同 Windows 设置 / 窗口焦点状态下行为不稳定，按 `SendInput` 模拟仍存在”按下没反应”的场景，故按题目要求直接移除该功能：
  - 输入栏右侧 🎤 按钮删除（`Views/MainWindow.xaml`），输入框单独占据宽度。
  - 删除 `MainWindow.xaml.cs#VoiceInput_Click` 与 `Infrastructure/NativeMethods.cs` 中的 `InvokeWindowsSpeechInput` / `INPUT` / `KEYBDINPUT` / `SendInput` 等 P/Invoke 代码，减少体积。
- 用户仍可在系统任意输入框中手动按 `Win+H` 使用 Windows 自带语音听写。

#### 2. 父子关系联动 & 整族消失动画

- **父子关系自动推断**：以 `IndentLevel` 与同分组内的 `OrderIndex` 顺序为唯一来源，每次”任何会变动层级或顺序”的操作后立即调用 `MainViewModel.RecomputeParents()`：
  - `ChangeIndent`（Tab / Shift+Tab / 右键缩进）
  - `AddTask`（新建任务）
  - `DropTask`（拖拽排序）
  - `ImportMarkdown`（导入）
  - 启动时一次性重算，修正旧数据。
  - 算法：分组内按 `OrderIndex` 顺序遍历，维护一个”层级栈”，遇到一项时弹出 ≥ 自己层级的项，栈顶即为父；`IndentLevel=0` 时 `ParentId=null`。
- **完成行为**（`MainViewModel.OnItemPropertyChanged`）：
  - 子待办勾选：若其父存在且未完成，则**只打钩、不移动到”已完成”分组**，并调用 `CheckParentCompletion` 检查父待办；
  - 父或独立待办勾选：调用 `MarkDescendantsCompletedSilently` 静默把所有后代设为完成（`_completingFamily` 标志抑制重入），随后抛出 `TaskCompleting` 事件给视图层；
  - 当父待办的所有直接子待办均已完成 → 自动把父待办勾上（递归触发上面的”父分支”，从而触发整族消失动画）。
- **整族消失动画**（`MainWindow.xaml.cs#OnTaskCompleting`）：
  - 收集 `Vm.CollectFamily(parent)`（父 + 所有递归子）；
  - 对每个仍在当前视图中的容器同时播放原有 1.1s 的”向左滑出 + 淡出 + 行高收起”动画，并叠加烟花特效；
  - 全部动画结束后回调 `Vm.FinishFamilyCompletion(parent)`，把整族一次性搬到”已完成”分组并刷新列表。
- 关键文件：`Models/TodoItem.cs`（`ParentId / IsCollapsed / HasChildren`）、`ViewModels/MainViewModel.cs`（`RecomputeParents / MarkDescendantsCompletedSilently / CollectFamily / FinishFamilyCompletion`）、`Views/MainWindow.xaml.cs#OnTaskCompleting`。

#### 3. 父待办折叠

- `TodoItem` 新增持久化属性 `IsCollapsed`，运行时属性 `HasChildren`（由 `RefreshItems` 维护）。
- `MainViewModel.RefreshItems` 增加过滤：任何祖先链上有 `IsCollapsed=true` 的项都从可见列表里剔除（`IsHiddenByCollapsedAncestor`）。
- 任务模板新增折叠箭头按钮：`HasChildren=true` 才显示，`▼` 展开 / `▶` 折叠，绑定 `ToggleCollapseCommand`；右键菜单同步增加”折叠 / 展开子待办”项。
- 折叠状态写入 `data.json`，下次启动恢复。

#### 4. 贴边隐藏闪屏修复

- 原因诊断：贴边显示态下旧代码仅用”光标坐标在窗口外”就立即触发隐藏；动画期间 probe 仍然工作，鼠标位于贴附边附近时会被反向触发，造成隐藏 → 立刻显示 → 又隐藏的来回闪烁。
- 修复（`MainWindow.xaml.cs`）：
  - 新增 `_isDockAnimating` 标志，`HideToEdge` / `ShowFromEdge` 动画期间 probe 直接 `return`，动画结束后才允许判断。
  - `EdgeProbeTick` 显示态用双重判断：`IsMouseOver`（WPF 命中测试，鼠标确实落在窗口上）**或**屏幕坐标位于窗口 + 40px 缓冲区内，任一为真都视为”鼠标仍在 app 上”，重置防抖计数器、不隐藏。
  - 鼠标连续离开 `OutsideTickThreshold=5` 次（≈450ms）才执行 `HideToEdge`，避免边缘抖动或快速来回移动触发误隐藏。
  - 隐藏缓冲区从 20px 提升到 40px。
- 效果：鼠标在 app 上任意移动 / 滚动 / 操作 → 始终保持显示；鼠标真正离开（450ms 内未再回到窗口）才滑回隐藏。

#### 5. 优先级默认中、移除”无优先级”

- 移除 UI 中的”无优先级”选项：
  - `MainViewModel.PriorityOptions` 删除 `None` 项；
  - `TaskEditDialog.xaml` 优先级 `UniformGrid` 由 4 列改 3 列，删除 `PrioNone` 单选；`TaskEditDialog.xaml.cs` 默认回填中、`ReadPriority` 兜底中。
- 新任务默认 `Priority.Medium`：
  - `TodoItem.priority` 字段默认值；
  - `MainViewModel.NewTaskPriority` 字段默认 + `AddTask` 完成后的重置。
- 旧数据迁移：`MainViewModel.MigrateNonePriority` 启动时把所有 `Priority.None` 改为 `Priority.Medium`，避免历史任务在 UI 上显示空白色条。
- `PriorityToBrushConverter / PriorityToTextConverter` 对 `None` 的兜底也改为中（橙色 / “中”）。

### 兼容性 & 体积

- 依赖与上次一致（`gong-wpf-dragdrop` + `H.NotifyIcon.Wpf` + `CommunityToolkit.Mvvm`），未引入新包。
- 移除语音相关 P/Invoke 后，`Infrastructure/NativeMethods.cs` 体积更小。
- 单文件 self-contained 发布产物 ~64 MB（启用单文件压缩 + 仅英文卫星资源 + 去调试符号），可在干净的 Windows 上免装 .NET 8 运行时直接运行。
- 构建命令：`build.bat` 或 `build.ps1`。

### 输出

- 单文件 exe 路径：`MinimalTodoApp/bin/Release/net8.0-windows/win-x64/publish/MinimalTodoApp.exe`

## 第 3 次优化（2026-05-30）

### 原始提示词

> 1、贴边隐藏功能在上面和左边生效了，但是在右边的时候没有生效，需要贴边右边的时候也生效
> 2、侧边栏的图标大小和文件大小不统一，需要图标大小统一，文件大小统一
> 3、新增默认分组，所有待办，所有未完成的待办都会显示在所有待办里面，所有待办分组初始化时在全部任务下面。初始化时为 全部任务 所有待办 工作 学习 生活 已完成
> 4、侧边栏缩小太宽了，缩小1/4，顶部栏太宽了，缩小1/4，输入栏太宽了，缩小1/4，并且输入栏支持手动调整大小，并且记忆调整的位置
> 5、开机启动需要记录下当前软件的位置，如果位置移动了，删除之前位置的启动项目，更新为当前位置的启动项目，确保软件挪了位置启动了，第一次是默认开机启动
> 6、添加任务的优先级、时间、提醒、父待办，改为 优先级、截至时间、周期提醒、父待办

> 截至时间不要下拉框
> 时间 改为一个选择时间的按钮，点击后弹出截至时间选项，按照下面，同类的在一行
> 分钟：1、10、30
> 小时：1、2、5
> 天：1、2、5
> 周：1、2、4
> 自定义时间：点击后自己选择时间，可以选择到小时分钟
> 周期提醒同理不要下拉框
> 改为一个选择时间的按钮，点击后弹出周期提醒时间选项，按照下面，同类的在一行
> 分钟：1、10、30
> 小时：1、2、5
> 天：1、2、5
> 周：1、2、4
> 自定义时间：可以输入文本 右边选择单位

### 实现要点

#### 1. 右侧贴边隐藏（DPI 修复）

- 原因诊断：贴边判定 `Left + Width >= wa.Right - SnapThresholdPx` 中，`Left/Width` 是 WPF DIP，而 `wa.Right` 来自 `MONITORINFO.rcWork` 物理像素。100% DPI 两者相等，>100% 时 `wa.Right` 远大于 `Left + Width`，条件永远不成立，因此“右侧贴边”从未生效；上 / 左只比 `wa.Top/Left=0`，恰好两套坐标系都是 0，所以一直正常。
- 修复（`Infrastructure/NativeMethods.cs`）：
  - 新增 `ToDip(Rect/Point, Visual)`，利用 `PresentationSource.CompositionTarget.TransformFromDevice` 把物理像素转换为 DIP；
  - 新增 `GetCursorScreenWorkAreaDip(Visual)` / `GetCursorPointDip(Visual)`，与 WPF Left/Top 同坐标系。
- `Views/MainWindow.xaml.cs` 中 `RestoreDockOnLoad`、`TryDockAfterDrag`、`DockTo`、`EdgeProbeTick` 全部改用 DIP 版本，三条边的判定 / 探针都在同一坐标系下比较。

#### 2. 侧边栏图标 / 文字尺寸统一

- 之前展开态各按钮 Height=34、图标 FontSize 多为 15（折叠新建分组用 18 例外）、文字默认 12px；折叠态宽 52。
- 现在统一为：
  - 按钮 Height=28，Padding `6,0`；
  - 所有图标 `FontSize=13`、`Width=18`、水平居中；
  - 所有文字 `FontSize=12`；
  - 分组圆点 12×12 → 10×10；
  - 折叠态 `CollapsedRailWidth` 52 → 39。
- 标题栏按钮也按比例收紧：`CaptionButton` 44×32 → 36×26、`CaptionToggleButton` 40×32 → 32×26。

#### 3. 默认「所有待办」聚合分组

- `Models/TodoGroup.cs` 新增 `IsAllUncompletedGroup` 标志位（与 `IsCompletedGroup` 同级，均为内置视图分组）。
- `MainViewModel`：
  - `SeedDefaultsIfEmpty` 新顺序 `所有待办 → 工作 → 学习 → 生活 → 已完成`（工作 / 学习 / 生活 顺序按用户要求调整）。
  - 新增 `EnsureAllUncompletedGroup` 迁移：旧数据加载时若没有“所有待办”分组则自动插入到首位并重排 `OrderIndex`。
  - `RefreshItems` 选中“所有待办”时 `Where(i => !i.IsCompleted)`（不再按 GroupId 过滤）；
  - `RefreshGroupCounts` 为“所有待办”单独按全部未完成总数计数；
  - `AddTask` / `MoveTaskToGroup` / `DeleteGroup` / `ClearGroup` 全部把“所有待办”视为不可承载真实任务的视图分组；
  - `FirstNormalGroup`、`MoveTargetGroups`、Markdown 导入的目标候选都排除“所有待办”，避免误把任务写入其中。
- 旧数据兼容：第一次打开新版会自动插入“所有待办”分组（蓝紫色 `#6366F1`），不影响既有任务归属。

#### 4. UI 缩窄 1/4 + 输入栏可调整并记忆

- 顶部栏：`WindowChrome.CaptionHeight` 40→30，外层 Grid `RowDefinition Height=40` 也跟着改成 30。
- 侧边栏：默认宽 150→113（`AppData.SidebarWidth` 默认值 + `MainWindow.xaml.cs#DefaultExpandedWidth`），最小宽 110→82；折叠态 52→39。
- 输入栏：
  - `AppData` 新增 `InputBarHeight`（默认 40），`MainViewModel.InputBarHeight` 双向绑定 + `partial OnInputBarHeightChanged` 触发持久化；
  - XAML 把任务区改成 5 行布局：在原 Row3 输入栏之上插入 `Auto` 高度的 `GridSplitter`（`InputBarSplitter`），输入栏所在 `InputBarRow` 改为定值高度（默认 40，MinHeight=28）；
  - `MainWindow.xaml.cs#ApplyInputBarHeight` 启动时把 VM 高度应用到行；`InputBarSplitter` 拖动结束触发 `SyncInputBarHeightBack`，把当前 `ActualHeight` 写回 VM 持久化；
  - 文本框改为 `TextWrapping=Wrap`，长标题在拉高的输入栏里自动换行，回车仍提交。

#### 5. 开机启动跟随当前 exe 位置

- 之前 `EnsureFirstRunStartup` 仅首启时写一次注册表，之后即便 exe 挪到别处，Run 项里还是旧路径，开机会去找旧位置（可能已被删除）。
- `Infrastructure/StartupManager.cs` 新增：
  - `GetRegisteredPath()` 读取并去掉首尾引号，返回当前 Run 项里记录的 exe 路径；
  - `SyncRegisteredPath()` 若已启用且路径与 `Environment.ProcessPath` 不一致，先 `DeleteValue` 再以新引号路径写回，确保只剩当前位置这一条。
- `MainViewModel.EnsureFirstRunStartup` 改造：首次启动仍 `SetEnabled(true)`；后续每次启动都调用 `SyncRegisteredPath()`，用户挪了 exe 位置、下次启动一次就完成自我搬迁。
