## 一、背景:这个项目是什么

- 原版是 C#/WPF + .NET 8 的 Windows 本地待办应用(单文件 exe 63MB),**完整保留在 `legacy/` 目录**,是一切迁移工作的参照物。
- 2026-06-12 起按用户指定技术栈整体重写:**Tauri v2 + Rust(rusqlite/SQLite WAL)+ React 19 + TypeScript + Zustand v5(无路由)+ Tailwind v4 + pragmatic-drag-and-drop + auto-animate**。包体约 10MB。
- 重写按里程碑 M1~M8 完成了主体(工程/任务/导入/主题/外壳/便签日历/更新/文档),之后又按用户逐轮反馈大量打磨(主题改版三次、瀑布流看板、日历三视图、贴边停靠对齐旧版等)。全部历史见 `优化记录.md` 与 git log。
- 架构、命令、约定、踩坑(time crate 钉 0.3.47、vite8 配套、IPC 三态约定等)都在 **`CLAUDE.md`**,不重复。

## 二、必须遵守的两个记录文档(用户的硬性仪式)

1. **`优化记录.md`(项目根)** — 每轮需求做完后追加记录,**必须包含用户原始提示词逐字**、改动摘要、commit hash。格式照现有条目。
2. **`release.md`(项目根)** — 发版时记版本号 + 变更摘要(最新在上)。版本号需三处同步递增:`src-tauri/tauri.conf.json`、`src-tauri/Cargo.toml`、`package.json`;面向用户的双语说明另写根 `release-notes.md`。

其他每轮规矩:i18n 新键必须 zh/en 双语同步(`src/lib/i18n.ts` 的 EXTRA 或 `src/i18n/*.json`);`npm run build` 必须过;改 Rust 跑 `cargo check`;中文 commit;**默认只本地提交,不 push、不跑 release.ps1,除非用户明说**。

## 四、下一步任务:继续旧版功能迁移

参照物全在 `legacy/MinimalTodoApp/`。以下是**尚未迁移**的旧版功能清单(按建议优先级),每项附旧版源码位置:

### 高优先级(日常使用明显缺失)
1. **任务编辑对话框**(`Views/TaskEditDialog.xaml(.cs)`):点击任务弹出完整编辑(标题/备注?/截止/优先级/提醒/所属标签)。新版目前只有内联改标题 + 右键菜单零散改属性。
2. **删除确认**(`Views/ConfirmDialog.xaml`):新版删除任务/标签/便签都是直接删,无确认、无撤销。旧版有 ConfirmDialog;todo-flow 参考项目用的是「toast + 撤销」也可。
3. **标签改颜色/图标**(`Views/ColorPicker.xaml`、`Views/IconPickerDialog.xaml`、`Infrastructure/GroupIcons.cs`):新版标签颜色只有默认蓝,无 UI 可改;旧版右键标签可改颜色/图标(S.Group.ChangeColor / S.Group.ChangeIcon 键已在词典里)。
4. **清空已完成**(S.Group.ClearCompleted 键):旧版已完成分组可一键清空。

### 中优先级
5. **新任务快捷提醒**:旧版输入栏可在新建时直接设周期提醒(QuickAdd 目前只有截止+优先级);键 S.SetAsReminder / S.ChooseReminder。
6. **便签增强**(`Views/NotesView.xaml.cs`、`Infrastructure/NoteImageStore.cs`、`MarkdownService.cs`):便签图片粘贴/存储(%AppData%\MinimalTodoApp\note-images 已有用户数据!)、便签拖拽排序/拖入分组(旧 NotesDropHandler)、旧版 `<color=#xxx>` 自定义标记的渲染兼容(用户便签里实际存在这种标记,现按原文显示)。
7. **帮助对话框**(`Views/HelpDialog.xaml`):S.Help.* 全套键已导入词典(任务/窗口/主题/更新各节),做个 Modal 渲染即可。
8. **导入/导出**:旧版 ☰ 菜单有数据导入导出(JSON)。

### 低优先级 / 锦上添花
9. **完成音效用旧版音乐**(`Infrastructure/CelebrationSound.cs`):现为 WebAudio 合成三连音;旧版是一段庆祝音乐。
10. **提醒 Toast 行为对齐**(`Views/ToastWindow.xaml`):旧版气泡在左下角、可点击聚焦任务;现为应用内顶部 Toast。
11. **托盘菜单随语言即时重建**(现启动时按语言构建一次)。
12. **单实例接管提示**:旧版二开时弹框征询;现直接唤起已有实例(可能够用,问用户)。

### 已知技术债
- `src/lib/themes.ts` 的 `isGlassTheme`、Rust 端 `set_acrylic`/`get_custom_themes` 等命令与 custom_themes 表为旧主题方案遗留,现 UI 未使用(保留无害,可问用户是否要恢复自定义主题编辑器)。
- 日历打开时窗口右扩逻辑(`src/App.tsx` 的 scheduleOpen effect)与贴边停靠存在潜在竞态(旧版注释专门提过),如出怪 bug 先查这两处交互。
- 自动更新检查在用户网络下连不通 GitHub(api.github.com),行为是优雅降级提示;发布链路(release.ps1)未实测。
